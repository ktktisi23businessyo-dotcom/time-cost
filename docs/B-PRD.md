# B. PRD（要件定義）

> 何を作るかの仕様。

---

## 1. 機能要件（画面/機能×入出力）

### 1-1 固定費登録

| 項目 | 内容 |
|------|------|
| **入力** | カテゴリ（必須）、項目名（「その他」の場合必須）、月額金額（必須・整数円）、メモ（任意） |
| **カテゴリ選択肢** | 家賃、保険、通信費、サブスク、車関連、その他 |
| **処理** | バリデーション → localStorage保存 → 合計再計算 → UI更新 |
| **出力** | 一覧への追加表示、時間コストの再計算表示 |

### 1-2 固定費一覧

| 項目 | 内容 |
|------|------|
| **表示** | カテゴリ、項目名、月額金額、メモ（任意） |
| **操作** | 各項目に編集ボタン、削除ボタン |

### 1-3 固定費編集

| 項目 | 内容 |
|------|------|
| **入力** | 既存データの編集フォーム |
| **処理** | 保存 → 再計算 |
| **出力** | 一覧・合計・時間コストの更新 |

### 1-4 固定費削除

| 項目 | 内容 |
|------|------|
| **入力** | 削除ボタンクリック |
| **処理** | 削除確認表示 → 削除実行 → 再計算 |
| **出力** | 一覧からの削除、合計・時間コストの更新 |

### 1-5 時間コスト計算（中核機能）

| 項目 | 内容 |
|------|------|
| **計算式** | 1日 = 月額÷30、1時間 = 1日÷24、1分 = 1時間÷60 |
| **表示** | 月額固定費、1日コスト、1時間コスト、1分コスト |
| **単位** | 円、小数点第2位まで（四捨五入） |

### 1-6 データ保存・復元

| 項目 | 内容 |
|------|------|
| **保存** | localStorage に固定費データ配列を保存 |
| **復元** | ページロード時に localStorage 読み込み、存在すれば UI 復元 |

---

## 2. 非機能要件

| 項目 | 要件 |
|------|------|
| **性能** | 固定費100件でも高速動作 |
| **ユーザビリティ** | 説明なしで操作可能 |
| **セキュリティ** | データはサーバー保存しない |
| **可用性** | 主要ブラウザで動作（PC・スマホ） |
| **レスポンシブ** | スマホで操作可能なレイアウト |

---

## 3. 制約条件

| 項目 | 内容 |
|------|------|
| **技術スタック** | HTML / CSS / JavaScript（フレームワーク不使用） |
| **データ保存** | localStorage のみ |
| **デプロイ** | GitHub Pages |
| **PWA** | manifest.json、Service Worker、HTTPS 必須 |

---

## 4. 受け入れ条件（テスト観点）

| 観点 | 期待結果 |
|------|----------|
| **固定費登録時** | 一覧更新、合計更新、時間コスト更新 |
| **再読み込み** | データ保持 |
| **不正入力** | エラー表示 |
| **PWA** | Chrome でインストール可能、アイコン表示、独立ウィンドウ起動 |
| **スマホ** | スマホブラウザ・PWA で操作可能 |
