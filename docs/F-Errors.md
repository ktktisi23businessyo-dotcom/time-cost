# F. エラーズ（開発中発生エラー記録）

> 作成途中で発生したエラーを記録する。再発防止・調査の参照用。

---

## 記録フォーマット

| 項目 | 説明 |
|------|------|
| **No** | 通し番号 |
| **日付** | 発生日時 |
| **Phase** | 発生したフェーズ（0/1/2/3） |
| **作業内容** | 何をしていたときか |
| **エラー内容** | 表示されたメッセージ・症状 |
| **原因** | 判明した原因（不明の場合は「調査中」） |
| **対応** | 実施した対応・解決方法 |
| **状態** | 未解決 / 解決 / 保留 |

---

## エラー一覧

| No | 日付 | Phase | 作業内容 | エラー内容 | 原因 | 対応 | 状態 |
|----|------|-------|----------|------------|------|------|------|
| 1 | - | 1 | 削除確認モーダル | 「削除する」「キャンセル」ボタンが選択できない | モーダル構造・pointer-events の影響 | モーダルを body 直下に移動、backdrop と content を分離、aria-hidden で表示制御、z-index 調整 | 解決 |

## 補足メモ（任意）

- エラー発生時はスクリーンショットやスタックトレースを添付すると調査しやすい
- 解決後も記録を残すことで、同様の事象が起きた際の参照になる
