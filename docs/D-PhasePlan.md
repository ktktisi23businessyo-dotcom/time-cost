## 2. 各フェーズの目的 / 成果物 / 完了条件

### Phase 0: 準備

| 項目 | 内容 |
|------|------|
| **目的** | 開発基盤の整備、仕様の確定 |
| **成果物** | 設計ドキュメント（A〜D）、ディレクトリ構成、Git リポジトリ |
| **完了条件** | docs/ に A〜D が揃い、開発方針が明確である |

### Phase 1: コア実装

| 項目 | 内容 |
|------|------|
| **目的** | 固定費管理と時間コスト表示の動作実装 |
| **成果物** | index.html、style.css、app.js、storage.js、calculator.js、ui.js |
| **完了条件** | 登録・編集・削除・計算・localStorage 保存・復元がすべて動作する。スマホで操作可能。 |

### Phase 2: PWA化

| 項目 | 内容 |
|------|------|
| **目的** | PWA としてインストール可能な状態にする |
| **成果物** | manifest.json、sw.js、アイコン（192x192, 512x512） |
| **完了条件** | Chrome でインストール可能、独立ウィンドウ起動、アイコン表示 |

### Phase 3: 公開

| 項目 | 内容 |
|------|------|
| **目的** | 本番環境での公開と動作確認 |
| **成果物** | GitHub Pages で公開された URL |
| **完了条件** | HTTPS でアクセス可能、PWA インストール可能、スマホで操作可能、受け入れ条件を満たす |
